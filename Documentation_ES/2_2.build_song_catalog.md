## 📋 Descripción General

Este script es el **segundo componente de la Fase 2** en el sistema de inteligencia de Charts de YouTube. Construye y mantiene un **catálogo canónico de canciones** extrayendo pares distintos (artista, canción) de las bases de datos semanales de charts y almacenándolos en una base de datos SQLite desduplicada y de crecimiento incremental.

**Mejora clave:** El script ahora integra el **sistema de ponderación de colaboraciones** (anteriormente en el Script 3) para resolver el país y el macro-género de cada canción **una sola vez**, en el momento de inserción en el catálogo. Esto elimina el procesamiento redundante en múltiples apariciones en los charts.

### Características Principales

- **ETL Idempotente**: Seguro de ejecutar múltiples veces; nunca crea registros duplicados
- **Resolución Única de País/Género**: Cada canción se procesa exactamente una vez usando el algoritmo de ponderación de colaboraciones
- **Claves Surrogadas Auto-incrementales**: IDs secuenciales para integridad referencial
- **Desduplicación por Clave Natural**: Usa la clave compuesta `(artist_names, track_name)` para unicidad
- **Integración de Metadatos de Artistas**: Resuelve países y géneros desde `artist_countries_genres.db` (Script 2_1)
- **Algoritmo de Ponderación de Colaboraciones**: Determina inteligentemente el país y género para pistas con múltiples artistas
- **Reparación de Registros Históricos**: Rellena automáticamente datos faltantes de país/género para entradas existentes en el catálogo
- **Detección Automática de Fuente**: Encuentra la base de datos semanal más reciente por identificador de semana ISO
- **Reporte de Progreso**: Muestra el progreso de inserción en 25%, 50%, 75% y 100%
- **Conexión de Solo Lectura a la Fuente**: Previene la modificación accidental de los datos fuente
- **Estadísticas del Catálogo**: Muestra métricas de crecimiento, distribución de país/género y artistas principales
- **Optimizado para CI/CD**: Cero dependencias externas; usa solo la biblioteca estándar de Python

### **Diagrama 3: Sistema de Ponderación de Colaboraciones**

<img src="https://raw.githubusercontent.com/adroguetth/Music-Charts-Intelligence/main/Documentation_ES/Diagramas/2_2.build_song_catalog/3.png" alt="Diagrama 3: Sistema de Ponderación de Colaboraciones" width="500">

Este diagrama muestra el **motor de decisión inteligente** para pistas con múltiples artistas:

1. **Entrada**: Recibe lista de artistas con sus países y géneros desde el diccionario de búsqueda
2. **Verificación de Artista Único**: Si solo hay 1 artista → retorna el país y género de ese artista inmediatamente ✅
3. **Filtrar Artistas Conocidos**: Eliminar artistas con país NULL/Unknown
4. **Contar Países**: Calcular la frecuencia de cada país entre los artistas conocidos
5. **Aplicar Reglas de Decisión**:

   | Regla                                     | Condición                       | Resultado                                                  |
   | :---------------------------------------- | :------------------------------ | :--------------------------------------------------------- |
   | **Regla 1 – Mayoría Absoluta**            | >50% del mismo país             | Retornar país mayoritario + inferir género desde jerarquía |
   | **Regla 2 – División Exacta 50/50**       | =50% con exactamente 2 países   | Retornar país mayoritario + inferir género                 |
   | **Regla 3 – Mayoría Relativa (<50%)**     | Mismo continente y ≤2 países    | Retornar país mayoritario + inferir género                 |
   | **Regla 4 – Fallback (multi-continente)** | <50% con diferentes continentes | Retornar "Multi-country" + "Multi-genre"                   |
   | **Regla 5 – Sin artistas conocidos**      | Todos los países NULL/Unknown   | Retornar "Unknown" + "Pop"                                 |

6. **Inferencia de Género** (`infer_genre_by_country`):
   - Recupera la jerarquía de géneros del país desde `GENRE_HIERARCHY`
   - Si un solo género tiene >50% entre los artistas → usarlo
   - De lo contrario, elegir el género de mayor rango presente en los géneros conocidos
   - Fallback al primer género de la jerarquía

7. **Salida**: Retorna la tupla final `(country, genre)` para la pista

### **Diagrama 4: Lógica de Inserción Idempotente con Resolución de País/Género**

<img src="https://raw.githubusercontent.com/adroguetth/Music-Charts-Intelligence/main/Documentation_ES/Diagramas/2_2.build_song_catalog/4.png" alt="Diagrama 4: Lógica de Inserción Idempotente con Resolución de País/Género" width="500">

Este diagrama detalla la **estrategia de desduplicación mejorada**:

1. **Inicio**: Recibir `artist_names` y `track_name` desde la fuente
2. **Verificación de Existencia**: Consultar la tabla `artist_track` para la clave natural coincidente
3. **Si el Registro Existe**:
   - Verificar si `artist_country` o `macro_genre` es NULL/Unknown
   - Si está incompleto → **ACTUALIZAR** con valores resueltos (reparación histórica)
   - Si está completo → **OMITIR** (incrementar contador de omitidos)
4. **Si el Registro No Existe**:
   - Analizar la lista de artistas usando delimitadores (`&`, `feat.`, `ft.`, etc.)
   - Buscar cada artista en el diccionario `artist_lookup`
   - Aplicar el algoritmo `resolve_country_and_genre()`
   - **INSERTAR** nuevo registro con valores resueltos
5. **Después de Todos los Registros**: Confirmar transacción y reportar estadísticas

**Garantía de Idempotencia**: Ejecutar el script múltiples veces con la misma fuente produce el mismo estado final del catálogo (sin duplicados, resolución única).

```python
    country, genre = row
    invalid_country = country is None or country.strip() == "" or country == "Unknown"
```

```yaml
  run: |
    echo "========================================"
    echo "🎵 INFORME DE CONSTRUCCIÓN DEL CATÁLOGO"
    echo "========================================"
    TOTAL=$(sqlite3 ... "SELECT COUNT(*) FROM artist_track;")
    COMPLETE=$(sqlite3 ... "SELECT COUNT(*) FROM artist_track 
      WHERE artist_country NOT IN ('Unknown', '') 
        AND macro_genre NOT IN ('Unknown', '');")
    MULTI=$(sqlite3 ... "SELECT COUNT(*) FROM artist_track WHERE artist_country = 'Multi-country';")
    
    echo "🎵 Canciones únicas totales: $TOTAL"
    echo "✅ Registros completos: $COMPLETE ($(( COMPLETE * 100 / TOTAL ))%)"
    echo "🌐 Colaboraciones multi-país: $MULTI ($(( MULTI * 100 / TOTAL ))%)"
    echo "========================================"
```



### Programación Cron

```cron
'15 13 * * 1'  # Minuto 15, Hora 13, Cualquier día del mes, Cualquier mes, Lunes
```
- **Ejecución**: Cada lunes a las 13:15 UTC
- **Desfase**: 1 hora y 15 minutos después del Script 1 (12:00 UTC)
- **Dependencia**: Requiere que el Script 2.1 (`artist_countries_genres.db`) se haya completado exitosamente

### Disparadores de Ejecución

Este workflow se ejecuta **solo** en:

- **Ejecución programada**: Cada lunes a las 13:15 UTC
- **Ejecución manual**: Mediante `workflow_dispatch` desde la interfaz de GitHub Actions

> **Nota**: La ejecución automática en `git push` ha sido desactivada. Los cambios en scripts o bases de datos no activan este workflow automáticamente. Para probar cambios, use la ejecución manual o espere la próxima ejecución programada.

## 🐛 Solución de Problemas

### Problemas Comunes y Soluciones

| Error                              | Causa Probable                     | Solución                                                     |
| :--------------------------------- | :--------------------------------- | :----------------------------------------------------------- |
| `Artist database not found`        | Script 2_1 no se ha ejecutado      | Ejecutar `2_1.artist_countries_genres.py` primero            |
| `Source directory does not exist`  | Script 1 no se ha ejecutado        | Ejecutar `1_download.py` primero                             |
| `No database files found`          | No hay bases de datos semanales    | Verificar `charts_archive/1_download-chart/databases/`       |
| `Missing 'chart_data' table`       | Base de datos fuente corrupta      | Re-ejecutar Script 1 para esa semana                         |
| `Database is locked`               | Acceso concurrente                 | Esperar o reiniciar; no hay bloqueos de larga duración       |
| `sqlite3.OperationalError`         | Permisos de archivo                | Verificar permisos de escritura en el directorio destino     |
| `NULL artist_country after repair` | Artista no está en DB de metadatos | Actualizar `artist_countries_genres.db` con el artista faltante |

## 🧪 Limitaciones Conocidas

- **Sin Índices**: No hay índices en `artist_country` o `macro_genre` para consultas analíticas más rápidas (aceptable para la escala actual)
- **Procesamiento de Fuente Única**: Procesa solo la base de datos más reciente por ejecución
- **Sin Modo Incremental**: Siempre procesa todos los registros de la fuente (no solo semanas nuevas)
- **Coincidencia de Nombres de Artistas**: Solo normalización exacta de cadenas; sin coincidencia difusa para errores ortográficos
- **Dependencia de Jerarquía de Géneros**: La precisión depende de la completitud de `GENRE_HIERARCHY` para cada país
