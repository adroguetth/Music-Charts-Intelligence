## 📋 General Description

This project is the fourth component of the YouTube Charts intelligence system. It takes the enriched weekly chart database (from Script 3) and **automatically generates professional Jupyter notebooks** with comprehensive visual analysis and AI-generated insights in both English and Spanish.

The system produces two fully-executed notebooks per week containing **25+ visualizations**, **12 analysis sections**, and **AI-powered commentary** generated by DeepSeek API, all cached to avoid redundant API calls.

### Key Features

- **25+ Visualizations**: Static charts using matplotlib, seaborn, and squarify (no JavaScript required)
- **AI-Generated Insights**: DeepSeek API provides intelligent commentary for each analysis section
- **Bilingual Output**: Generates both English and Spanish notebooks from a single run
- **Smart Caching**: Per-week, per-language cache system avoids redundant API calls
- **Weekly Automation**: Fully automated via GitHub Actions, runs every Monday
- **Sliding Window Retention**: Keeps only last 6 notebooks per language automatically
- **Executive Summary**: 30-line detailed summary with actionable recommendations
- **YouTube-Style Styling**: Visualizations use YouTube's color palette (#FF0000, #F9F9F9, etc.)
- **Zero JavaScript**: All charts are static matplotlib/seaborn, compatible with GitHub preview
- **Database Agnostic**: Works with any enriched database from Script 3

### **Diagram 1: Main Flow Overview**

<img src="https://drive.google.com/uc?export=view&id=XXXXXXXXXXXXXXXXXX" alt="Main Flow Overview" width="400">

This diagram shows the **high-level pipeline** of the entire system:

1. **Input**: Reads enriched weekly database (`youtube_charts_YYYY-WXX_enriched.db`)
2. **Data Loading**: Loads data, calculates engagement, extracts temporal features
3. **Data Summaries**: Pre-calculates metrics for AI prompts (top countries, genre engagement, etc.)
4. **Cache Check**: For each section, checks if insight exists in cache
   - **If cached** → Uses cached insight ✅
   - **If not cached** → Queries DeepSeek API → Saves to cache
5. **Notebook Generation**: Creates Jupyter notebook with code cells and AI insights
6. **Notebook Execution**: Runs `nbconvert --execute` to embed all outputs
7. **Cleanup**: Removes old notebooks (keeps last 6 per language)
8. **GitHub Commit**: Automatically commits and pushes changes

### **Diagram 2: AI Insights Caching System**

<img src="https://drive.google.com/uc?export=view&id=XXXXXXXXXXXXXXXXXX" alt="Caching System" width="350">

This diagram details the **intelligent caching mechanism**:

1. **Section Request**: System needs insight for a specific section (e.g., "genre_engagement")
2. **Data Fingerprint**: Generates MD5 hash of relevant data subset:
   - `general_stats` → hash of (total_songs, total_views, total_likes)
   - `top_countries` → hash of top 10 countries string
   - `genre_engagement` → hash of genre engagement rates
3. **Cache Lookup**: Checks JSON file for key `{section}_{hash}`
   - **Cache hit** → Returns insight immediately (no API call)
   - **Cache miss** → Proceeds to DeepSeek API
4. **DeepSeek Query**: Sends language-specific prompt with data summary
5. **Insight Storage**: Saves result with hash key for future runs
6. **Cache Isolation**: Separate cache files per week and language
   - `youtube_charts_2026-W14_en.json`
   - `youtube_charts_2026-W14_es.json`

### **Diagram 3: Notebook Generation Pipeline**

<img src="https://drive.google.com/uc?export=view&id=XXXXXXXXXXXXXXXXXX" alt="Notebook Generation" width="350">

This diagram shows how the **Jupyter notebook is constructed**:

1. **Template Loading**: Section titles and structure defined in `get_section_titles()`
2. **Cell Creation**: For each section, creates markdown and code cells:
   - **Markdown cells**: Section headers and AI insights
   - **Code cells**: Python code for analysis and visualizations
3. **Dynamic Content Injection**:
   - Database path (relative path from notebook to repo root)
   - Week/year information
   - AI insights (from cache or API)
4. **nbformat Assembly**: Builds notebook using `nbformat.v4` structures
5. **File Writing**: Saves as `.ipynb` with proper metadata (Python 3 kernel)
