## 📋 General Description

This script is the **second component of Phase 2** in the YouTube Charts intelligence system. It builds and maintains a **canonical song catalog** by extracting distinct (artist, track) pairs from weekly chart databases and storing them in a deduplicated, incrementally growing SQLite database.

The script implements an **idempotent ETL pipeline** that can be run repeatedly without creating duplicate records. It automatically detects the most recent weekly database, extracts unique artist-track combinations, and inserts only new records into the catalog using a natural key deduplication strategy.

### Key Features

- **Idempotent ETL**: Safe to run multiple times; never creates duplicate records
- **Auto-incrementing Surrogate Keys**: Sequential IDs for referential integrity
- **Natural Key Deduplication**: Uses (artist_names, track_name) composite key for uniqueness
- **Automatic Source Detection**: Finds the most recent weekly database by ISO week identifier
- **Progress Reporting**: Shows insertion progress at 25%, 50%, 75%, and 100%
- **Read-Only Source Connection**: Prevents accidental modification of source data
- **Catalog Statistics**: Displays growth metrics, sample entries, and top artists
- **CI/CD Optimized**: Zero external dependencies; uses only Python standard library

9. **Generate Statistics**: Displays inserted count, skipped count, growth percentage, and sample entries

- Referential integrity for downstream analytics
- Stable identifiers even if artist/track names change
- Efficient JOIN operations with fact tables

### **Diagram 3: Idempotent Insertion Logic**

<img src="https://raw.githubusercontent.com/adroguetth/Music-Charts-Intelligence/main/Documentation_EN/Diagrams/2.2_build_song_catalog/EN-Diagram3.png" alt="Diagram 3: Idempotent Insertion Logic" width="700">

This diagram details the **deduplication strategy**:

1. **Start**: Receive artist_names and track_name from source
2. **Existence Check**: Query `artist_track` table for matching natural key
   - `SELECT 1 FROM artist_track WHERE artist_names = ? AND track_name = ? LIMIT 1`
3. **If Record Exists**: Skip insertion (increment skipped counter) → Continue to next record
4. **If Record Not Exists**: Insert new record with auto-generated ID (increment inserted counter) → Continue to next record
5. **After All Records**: Commit transaction and report statistics

**Idempotency Guarantee**: Running the script multiple times with the same source produces the same final catalog state (no duplicates, no data loss).

```python
def get_most_recent_database(directory: Path) -> Optional[Path]:
    """
    Identify the most recent YouTube Charts database file.
    
    Selection algorithm:
        1. Filter files matching pattern: youtube_charts_20YY-Www.db
        2. Extract ISO week identifiers for comparison
        3. Sort by year (descending), then by week number (descending)
        4. Return path to the most recent valid database
    """
```

**Filename Pattern**: `youtube_charts_2026-W15.db`

**Sorting Logic**:

- `2026-W15` > `2025-W52` (lexicographic order works due to ISO format)
- Correctly handles year boundaries

```sqlite
CREATE TABLE IF NOT EXISTS artist_track (

```

**AUTOINCREMENT Behavior**: Ensures SQLite generates monotonically increasing IDs and never reuses deleted IDs, which is important for referential integrity in downstream analytics.

#### **4. Record Existence Check**

```python
def record_exists(cursor: sqlite3.Cursor, artist_names: str, track_name: str) -> bool:
    """
    Check for existence of an artist-track pair in the catalog.
    
    Performs an efficient existence check using SELECT 1 with LIMIT 1
    to minimize I/O overhead while maintaining accuracy.
    """
    query = """
        SELECT 1
        FROM artist_track
        WHERE artist_names = ?
          AND track_name = ?
        LIMIT 1
    """
```

**Performance Optimization**: `LIMIT 1` stops scanning after the first match, reducing I/O overhead.

#### **5. Data Extraction Query**

```python
extract_query = """
    SELECT DISTINCT
        "Artist Names" AS artist_names,
        "Track Name" AS track_name
    FROM chart_data
    WHERE "Artist Names" IS NOT NULL
      AND "Track Name" IS NOT NULL
      AND TRIM("Artist Names") != ''
      AND TRIM("Track Name") != ''
    ORDER BY "Artist Names", "Track Name"
"""
```

**Filtering Logic**:

- Excludes NULL values
- Excludes empty strings after trimming whitespace
- Uses `DISTINCT` to reduce processing overhead for duplicates within source

#### **6. Idempotent Insertion Logic**

```python
insert_statement = """
    INSERT INTO artist_track (artist_names, track_name)
    VALUES (?, ?)
"""

if not record_exists(target_cursor, artist_names, track_name):
    target_cursor.execute(insert_statement, (artist_names, track_name))
    inserted_count += 1
else:
    skipped_count += 1
```

**Idempotency Guarantee**: Each (artist_names, track_name) pair appears at most once in the catalog, regardless of how many times the script runs.

#### **7. Progress Reporting**

```python
batch_size = 25
progress_interval = max(1, total_extracted // 4)  # Show progress at 25%, 50%, 75%
```

#### **8. Catalog Statistics**

```python
def get_catalog_statistics(connection: sqlite3.Connection) -> Tuple[int, int]:
    """Retrieve current catalog statistics."""
    cursor = connection.cursor()
    cursor.execute("SELECT COUNT(*), COALESCE(MAX(id), 0) FROM artist_track")
    total_records, latest_id = cursor.fetchone()
    return total_records, latest_id
```

**Return Values**:

- `total_records`: Number of unique songs in catalog
- `latest_id`: Highest auto-generated ID (useful for incremental processing)

## 🐛 Troubleshooting

### Common Issues and Solutions

| Error                             | Likely Cause              | Solution                                           |
| :-------------------------------- | :------------------------ | :------------------------------------------------- |
| `Source directory does not exist` | Script 1 hasn't run       | Run `1_download.py` first                          |
| `No database files found`         | No weekly databases       | Check `charts_archive/1_download-chart/databases/` |
| `Missing 'chart_data' table`      | Corrupted source database | Re-run Script 1 for that week                      |
| `Database is locked`              | Concurrent access         | Wait or restart; no long-running locks             |
| `sqlite3.OperationalError`        | File permissions          | Check write permissions on target directory        |
