## 📋 General Description

This script is the **second component** of the YouTube Charts intelligence system. It takes raw artist names extracted by Script 1 and **enriches them with geographic and genre metadata** by querying multiple open knowledge bases. The result is a structured, cumulative database of artists with their country of origin and primary music genre.

### Why a Modular Architecture?

The original script [`2_1.build_artist_db.py`](https://github.com/adroguetth/Music-Charts-Intelligence/blob/main/scripts/scripts_backup/2_1.build_artist_db_original.py) grew to **over 10,000 lines** , making it:

- **Difficult to maintain** (a single change could break multiple features)
- **Hard to test** (no isolation between components)
- **Slow to debug** (issues required tracing through massive functions)

The refactored version (`2_1.build_artist_db.py`) splits the code into **30+ focused modules**, each with a single responsibility:

| Problem                             | Solution                            |
| :---------------------------------- | :---------------------------------- |
| 10,000-line monolithic script       | 30+ small modules (<300 lines each) |
| API logic mixed with business logic | Separate `apis/` package            |
| Hard-coded country/genre data       | External `dictionaries/` package    |
| Duplicate code across functions     | Shared `utils/` package             |
| Difficult to add new data sources   | Plug-and-play API modules           |

### Key Features

- **Modular Architecture**: 30+ focused modules instead of a 10,000-line monolithic script
- **Multi-Source Lookup**: Intelligent cascading queries to MusicBrainz, Wikipedia, Wikidata
- **DeepSeek AI Fallback**: Uses DeepSeek API as last resort (cost-effective, ~$0.002 per 100 artists)
- **Smart Name Variation**: Generates up to 15 variations per artist for maximum match rate
- **Geographic Intelligence**: Country detection from 30,000+ terms (cities, demonyms, regions)
- **Genre Classification**: 200+ macro-genres and 5,000+ subgenre mappings with weighted voting
- **Country-Specific Rules**: Special handling for 50+ countries (K-Pop for South Korea, etc.)
- **Script Detection**: Automatic language detection for non-Latin scripts
- **Intelligent Updates**: Only fills missing data, never overwrites correct information
- **In-Memory Caching**: Avoids redundant API calls within a session
- **CI/CD Optimized**: Specifically configured for GitHub Actions

### 1. Main Orchestrator (`2_1.build_artist_db.py`)

**Purpose**: Entry point that coordinates all other modules. Only ~120 lines.

```python
#!/usr/bin/env python3
"""
Artist Country + Genre Detection System - Main Orchestrator
"""

import sys
from pathlib import Path

# Add parent directory to sys.path for module imports
SCRIPT_DIR = Path(__file__).parent.absolute()
PROJECT_ROOT = SCRIPT_DIR.parent
if str(PROJECT_ROOT) not in sys.path:
    sys.path.insert(0, str(PROJECT_ROOT))

# Imports from modular package
from build_artist_db.config import CHARTS_DB_DIR, ARTIST_DB_PATH, setup_logging
from build_artist_db.utils.db_utils import (
    create_database, artist_in_database, insert_artist,
    count_artists_in_database, get_latest_chart_database, get_artists_from_chart_db,
)
from build_artist_db.country_detector import search_country
from build_artist_db.genre_detector import search_artist_genre

logger = setup_logging()


def main() -> None:
    """Main execution function."""
    logger.info("🎵 Artist Country + Genre Detection System - MODULAR MODE")
    
    # 1. Initialize database
    create_database()
    
    # 2. Find latest chart database
    chart_db = get_latest_chart_database()
    if not chart_db:
        logger.error("❌ Could not obtain chart database")
        sys.exit(1)
    
    # 3. Extract unique artists
    chart_artists = get_artists_from_chart_db(chart_db)
    
    # 4. Process each artist
    for artist in sorted(chart_artists):
        exists, db_country, db_genre = artist_in_database(artist)
        
        if exists and db_country and db_genre:
            logger.info(f"  ✅ {artist} → {db_country} | {db_genre}")
            continue
        
        # Search for missing information
        country, country_source = search_country(artist)
        genre, genre_source = search_artist_genre(artist, country)
        
        # Update database
        insert_artist(artist, country or "Unknown", genre, f"{country_source} | {genre_source}")
    
    # 5. Final summary
    logger.info(f"📈 Total artists in database: {count_artists_in_database()}")


if __name__ == "__main__":
    main()
```



**What makes this module special:**

- **Only 120 lines** (vs 10,000+ in the original)

- **No API logic** (delegated to `apis/`)

- **No data processing** (delegated to `utils/`)

- **No hard-coded data** (delegated to `dictionaries/`)

- **Easy to test** (each dependency can be mocked)

#### 3.1 Text Utilities (`utils/text_utils.py`)

**Purpose**: String normalization, name variation generation, and script detection.

```python
"""
Text normalization and name variation generation utilities.
"""

import re
import unicodedata
from typing import List, Optional


def normalize_text(text: str) -> str:
    """Lowercase, collapse whitespace, strip punctuation."""
    if not text:
        return ""
    text = text.lower().strip()
    text = re.sub(r'\s+', ' ', text)
    text = text.strip('.,;:¿?¡!()[]"\'«»…–—')
    return text


def generate_name_variations(name: str) -> List[str]:
    """Generate basic variations: original, no accents, no dots, no hyphens."""
    variations = [name]
    
    # Remove accents
    no_accents = ''.join(c for c in unicodedata.normalize('NFD', name) 
                         if unicodedata.category(c) != 'Mn')
    if no_accents != name:
        variations.append(no_accents)
    
    # Remove dots
    no_dots = name.replace('.', '')
    if no_dots != name:
        variations.append(no_dots)
    
    # Replace hyphens with spaces
    no_hyphens = name.replace('-', ' ')
    if no_hyphens != name:
        variations.append(no_hyphens)
    
    return list(dict.fromkeys(variations))


ARTIST_PREFIXES = {'dj': ['DJ', 'Dj', 'dj'], 'mc': ['MC', 'Mc', 'mc'], 
                   'lil': ['Lil', 'lil', 'LIL'], 'the': ['The', 'the', 'THE'], ...}


def remove_artist_prefixes(name: str) -> List[str]:
    """Remove common prefixes (DJ, MC, Lil, The, etc.)."""
    variations = [name]
    for prefix_base, variants in ARTIST_PREFIXES.items():
        for variant in variants:
            if re.match(r'^' + re.escape(variant) + r'\s+', name):
                without_prefix = re.sub(r'^' + re.escape(variant) + r'\s+', '', name)
                if without_prefix:
                    variations.append(without_prefix)
    return list(dict.fromkeys(variations))


def generate_all_variations(name: str) -> List[str]:
    """Generate up to 15 variations combining all strategies."""
    all_vars = set()
    for var in generate_name_variations(name):
        all_vars.add(var)
        for no_prefix in remove_artist_prefixes(var):
            all_vars.add(no_prefix)
            for var2 in generate_name_variations(no_prefix):
                all_vars.add(var2)
    
    result = [name]
    for var in sorted(all_vars - {name}, key=len, reverse=True):
        if var and len(var) > 1:
            result.append(var)
    return result[:15]


def detect_script_from_name(name: str) -> Optional[str]:
    """Detect writing system and return ISO 639-1 language code."""
    # Devanagari (Hindi, Marathi, Nepali)
    if re.search(r'[\u0900-\u097F]', name):
        return 'hi'
    # Arabic / Urdu
    elif re.search(r'[\u0600-\u06FF]', name):
        if re.search(r'[گچپژ]', name):
            return 'ur'  # Urdu specific characters
        return 'ar'
    # Cyrillic (Russian, Ukrainian, Bulgarian, Serbian)
    elif re.search(r'[\u0400-\u04FF]', name):
        if re.search(r'[іїєґ]', name):
            return 'uk'
        if re.search(r'ъ', name):
            return 'bg'
        if re.search(r'[ћђџљњ]', name):
            return 'sr'
        return 'ru'
    # Japanese (Hiragana/Katakana)
    elif re.search(r'[\u3040-\u309F\u30A0-\u30FF]', name):
        return 'ja'
    # Chinese (Han) - distinguish from Korean
    elif re.search(r'[\u4E00-\u9FFF]', name):
        if re.search(r'[\uAC00-\uD7AF]', name):
            return 'ko'
        return 'zh'
    # Hangul (Korean)
    elif re.search(r'[\uAC00-\uD7AF]', name):
        return 'ko'
    # Latin with accents (Spanish/Portuguese)
    elif re.search(r'[áéíóúâêôàèìòùäëïöü]', name):
        if re.search(r'[çãõ]', name):
            return 'pt'
        return 'es'
    
    return None
```

**Why this module exists:**

- **Centralizes text processing** (used by country_detector, genre_detector, and APIs)

- **Name variations** improve match rates across different data sources

- **Script detection** guides Wikipedia language selection (Korean artists → Korean Wikipedia)

  

#### 3.2 Database Utilities (`utils/db_utils.py`)

**Purpose**: All SQLite operations in one place.

```python
"""
Database operations for the artist database.
"""

import sqlite3
import re
from pathlib import Path
from typing import Optional, Tuple, Set

from ..config import ARTIST_DB_PATH, CHARTS_DB_DIR, logger
from .artist_parser import split_artists


def create_database() -> None:
    """Create artist table if it doesn't exist."""
    conn = sqlite3.connect(str(ARTIST_DB_PATH))
    cursor = conn.cursor()
    cursor.execute('''
        CREATE TABLE IF NOT EXISTS artist (
            name TEXT PRIMARY KEY,
            country TEXT,
            macro_genre TEXT
        )
    ''')
    conn.commit()
    conn.close()
    logger.info("✅ Table 'artist' ready")


def artist_in_database(artist: str) -> Tuple[bool, Optional[str], Optional[str]]:
    """Check if artist exists and return (exists, country, macro_genre)."""
    conn = sqlite3.connect(str(ARTIST_DB_PATH))
    cursor = conn.cursor()
    cursor.execute('SELECT country, macro_genre FROM artist WHERE name = ?', (artist,))
    res = cursor.fetchone()
    conn.close()
    if res:
        return True, res[0], res[1]
    return False, None, None


def insert_artist(artist: str, country: str, genre: Optional[str] = None, source: str = "") -> None:
    """Insert new artist or update missing fields only."""
    conn = sqlite3.connect(str(ARTIST_DB_PATH))
    cursor = conn.cursor()
    cursor.execute('SELECT country, macro_genre FROM artist WHERE name = ?', (artist,))
    existing = cursor.fetchone()
    
    if existing:
        existing_country, existing_genre = existing
        update_country = country != 'Unknown' and (not existing_country or existing_country == 'Unknown')
        update_genre = genre and genre != 'Unknown' and (not existing_genre or existing_genre == 'Unknown')
        
        if update_country or update_genre:
            set_clauses = []
            params = []
            if update_country:
                set_clauses.append("country = ?")
                params.append(country)
            if update_genre:
                set_clauses.append("macro_genre = ?")
                params.append(genre)
            params.append(artist)
            cursor.execute(f"UPDATE artist SET {', '.join(set_clauses)} WHERE name = ?", params)
            conn.commit()
    else:
        cursor.execute("INSERT INTO artist (name, country, macro_genre) VALUES (?, ?, ?)", 
                      (artist, country, genre))
        conn.commit()
    
    conn.close()


def get_latest_chart_database() -> Optional[Path]:
    """Find most recent YouTube chart database by week identifier."""
    if not CHARTS_DB_DIR.exists():
        return None
    
    db_files = list(CHARTS_DB_DIR.glob("youtube_charts_*.db"))
    if not db_files:
        return None
    
    def week_key(file_path: Path) -> tuple:
        match = re.search(r'youtube_charts_(\d{4})-W(\d{1,2})\.db', file_path.name)
        if match:
            return (int(match.group(1)), int(match.group(2)))
        return (0, file_path.stat().st_mtime)
    
    latest = max(db_files, key=week_key)
    logger.info(f"📁 Using database: {latest.name}")
    return latest


def get_artists_from_chart_db(db_path: Path) -> Set[str]:
    """Extract all unique artist names from chart database."""
    artists = set()
    conn = sqlite3.connect(str(db_path))
    cursor = conn.cursor()
    
    # Find the artist column (may have different names)
    cursor.execute("PRAGMA table_info(chart_data)")
    columns = [col[1] for col in cursor.fetchall()]
    artist_column = None
    for col in ['Artist Names', 'Artist', 'Artists', 'artist', 'Artista']:
        if col in columns:
            artist_column = col
            break
    
    if artist_column:
        safe_column = f'"{artist_column}"' if ' ' in artist_column else artist_column
        cursor.execute(f"SELECT {safe_column} FROM chart_data WHERE {safe_column} IS NOT NULL")
        for row in cursor.fetchall():
            if row[0]:
                artists.update(split_artists(str(row[0])))
    
    conn.close()
    logger.info(f"📊 {len(artists)} unique artists found")
    return artists
```

**Why this module exists:**

- **Single responsibility** (only database operations)

- **Reusable** across the entire application

- **Easy to test** (can be mocked)

- **Centralized error handling** for database operations

  

#### 3.3 Artist Parser (`utils/artist_parser.py`)

```python
"""
Artist name parsing utilities.
"""

import re
from typing import List


def split_artists(artist_str: str) -> List[str]:
    """
    Split a string containing multiple artists into individual names.
    
    Handles: "feat.", "ft.", "&", "and", "y", commas, parentheses.
    
    Examples:
        "ROSÉ & Bruno Mars" → ["ROSÉ", "Bruno Mars"]
        "Bad Bunny feat. J Balvin" → ["Bad Bunny", "J Balvin"]
        "ROSÉ, Bruno Mars, Lady Gaga" → ["ROSÉ", "Bruno Mars", "Lady Gaga"]
    """
    if not artist_str or not isinstance(artist_str, str):
        return []
    
    # Normalize featuring patterns
    artist_str = re.sub(r'\s+feat\.?\s+', ', ', artist_str, flags=re.IGNORECASE)
    artist_str = re.sub(r'\s+ft\.?\s+', ', ', artist_str, flags=re.IGNORECASE)
    artist_str = re.sub(r'\s+featuring\s+', ', ', artist_str, flags=re.IGNORECASE)
    
    # Replace conjunctions
    artist_str = re.sub(r'\s+&\s+', ', ', artist_str)
    artist_str = re.sub(r'\s+y\s+', ', ', artist_str)
    artist_str = re.sub(r'\s+and\s+', ', ', artist_str, flags=re.IGNORECASE)
    
    # Remove parentheses (often contain feat. info)
    artist_str = re.sub(r'[\(\)]', '', artist_str)
    
    # Split by comma and clean
    artists = [a.strip() for a in artist_str.split(',')]
    artists = [a for a in artists if a and len(a) > 1]
    
    # Skip placeholders
    skip_words = {'various', 'various artists', 'unknown', 'varios', 'varios artistas'}
    artists = [a for a in artists if a.lower() not in skip_words]
    
    return artists
```

def search_musicbrainz_country_cached(artist: str) -> Tuple[Optional[str], str]:
    """Search MusicBrainz for artist's country of origin."""
    cache = get_cache()
    if artist in cache['musicbrainz_country']:
        return cache['musicbrainz_country'][artist]
    
    sessions = get_http_sessions()
    session = sessions['musicbrainz']
    
    try:
        url = "https://musicbrainz.org/ws/2/artist/"
        headers = {'User-Agent': 'ArtistDB/5.0 (contact@example.com)'}
        params = {'query': artist, 'fmt': 'json', 'limit': 5}
        resp = session.get(url, headers=headers, params=params, timeout=10)
        
        if resp.status_code != 200:
            return None, "Not found"
        
        data = resp.json()
        if not data.get('artists'):
            return None, "Not found"
        
        # Find best match using fuzzy matching
        artist_lower = artist.lower()
        best_score = 0.0
        best_artist = None
        
        for mb_artist in data['artists'][:5]:
            mb_name = mb_artist.get('name', '').lower()
            bonus = 0.3 if (artist_lower in mb_name or mb_name in artist_lower) else 0.0
            score = SequenceMatcher(None, artist_lower, mb_name).ratio() + bonus
            if score > best_score:
                best_score = score
                best_artist = mb_artist
        
        if best_score < 0.55:
            return None, "Not found"
        
        # Extract country from area or begin-area fields
        for field in ['area', 'begin-area']:
            if field in best_artist:
                country_raw = best_artist[field].get('name', '')
                if country_raw:
                    country_norm = validate_and_normalize_country(country_raw)
                    if country_norm:
                        result = (country_norm, f"MusicBrainz ({artist})")
                        cache['musicbrainz_country'][artist] = result
                        return result
        
        return None, "Not found"
        
    except Exception as e:
        logger.debug(f"MusicBrainz error for {artist}: {e}")
        return None, "Not found"

                if 'tags' in mb_data and not candidates:
                    for tag in mb_data['tags'][:5]:

        return _DEEPSEEK_CACHE[artist]
    
    client = _get_deepseek_client()
    if not client:
        return None, None, "DeepSeek not available"

    time.sleep(0.5)
    
    # Build prompt
    if context_country:

You are a music knowledge expert. Provide accurate information about the musical artist "{artist}".

The artist is known to be from {context_country} (but please verify if correct). Return ONLY a valid JSON object:
{{

}}

Be precise. If unsure, set to null. No additional text.
"""
    else:

You are a music knowledge expert. Provide accurate information about the musical artist "{artist}".

Return ONLY a valid JSON object:
{{
    "country": "country of origin",
    "genre": "primary musical genre"
}}

Be precise. If unsure, set to null. No additional text.
"""
    
    try:
        response = client.chat.completions.create(
            model="deepseek-chat",
            messages=[{"role": "user", "content": prompt}],
            temperature=0.1,
            max_tokens=200
        )
        content = response.choices[0].message.content.strip()
        
        # Extract JSON
        json_match = re.search(r'\{[^{}]*\}', content)

```python

Genre detection and voting system for artists.
"""

import time

from .config import logger

from .apis.musicbrainz import search_musicbrainz_genre_cached
from .apis.wikidata import search_wikidata_genre_cached

    search_wikipedia_infobox_genre_cached,
    search_wikipedia_summary_genre_cached,
)
from .apis.deepseek import search_deepseek_fallback
from .dictionaries.genres import GENRE_MAPPINGS
from .dictionaries.macro_genres import MACRO_GENRES, GENERIC_MACROS
from .dictionaries.country_rules import COUNTRY_GENRE_PRIORITY, COUNTRY_SPECIFIC_RULES, DEFAULT_GENRE_PRIORITY
from .dictionaries.stopwords import GENRE_STOPWORDS


def normalize_genre(genre_text: str) -> Tuple[Optional[str], Optional[str]]:
    """Map raw genre string to (macro_genre, subgenre)."""
    if not genre_text:
        return None, None
    
    text_norm = genre_text.lower().strip()
    
    # Skip stopwords
    if text_norm in GENRE_STOPWORDS:
        return None, None
    
    # Direct lookup
    if text_norm in GENRE_MAPPINGS:
        return GENRE_MAPPINGS[text_norm]
    
    # Substring match for longer variants
    for genre_variant in sorted(GENRE_MAPPINGS.keys(), key=len, reverse=True):
        if genre_variant in text_norm and len(genre_variant) > 3:
            return GENRE_MAPPINGS[genre_variant]
    
    return None, None


def select_primary_genre(artist: str, genre_candidates: List[Tuple[str, int, str]],
                         country: Optional[str] = None, detected_lang: Optional[str] = None) -> Optional[str]:
    """Select primary macro-genre using weighted voting system."""
    if not genre_candidates:
        return None
    
    macro_votes = defaultdict(float)
    
    for subgenre, weight, source in genre_candidates:
        macro, _ = normalize_genre(subgenre)
        if macro:
            final_weight = weight
            
            # Source-based bonuses
            if 'musicbrainz' in source:
                final_weight *= 1.5
            if 'infobox' in source:
                final_weight *= 1.2
            if 'wikidata' in source:
                final_weight *= 1.3
            
            # Term bonuses for specific genres
            sub_lower = subgenre.lower()
            if any(term in sub_lower for term in ['reggaeton', 'reggaetón', 'trap latino']):
                final_weight *= 1.4
                macro = 'Reggaetón/Trap Latino'
            if any(term in sub_lower for term in ['k-pop', 'kpop', 'korean pop']):
                final_weight *= 1.4
                macro = 'K-Pop/K-Rock'
            if any(term in sub_lower for term in ['sertanejo', 'funk brasileiro']):
                final_weight *= 1.4
            
            macro_votes[macro] += final_weight
    
    # Apply country priority bonuses
    if country and macro_votes:
        priority = COUNTRY_GENRE_PRIORITY.get(country, DEFAULT_GENRE_PRIORITY)
        for macro in list(macro_votes.keys()):
            if macro in priority:
                idx = priority.index(macro)
                if idx == 0:
                    macro_votes[macro] *= 2.0
                elif idx == 1:
                    macro_votes[macro] *= 1.5
                else:
                    macro_votes[macro] *= 1.2
    
    # Apply country-specific rules
    if country and macro_votes and country in COUNTRY_SPECIFIC_RULES:
        rule = COUNTRY_SPECIFIC_RULES[country]
        if "force_macro" in rule:
            for macro in list(macro_votes.keys()):
                if macro != rule["force_macro"]:
                    peso = macro_votes.pop(macro, 0)
                    macro_votes[rule["force_macro"]] += peso * rule.get("bonus_extra", 1.0)
    
    if macro_votes:
        return max(macro_votes.items(), key=lambda x: x[1])[0]
    
    return None

    """Determine primary genre using cascading sources."""
    all_candidates = []
    variations = generate_all_variations(artist)

    # 1. MusicBrainz
    for var in variations[:2]:
        candidates = search_musicbrainz_genre_cached(var)
        all_candidates.extend(candidates)
        if candidates:
            break
        time.sleep(0.5)
    
    # 2. Wikidata
    if not all_candidates:
        for var in variations[:2]:
            candidates = search_wikidata_genre_cached(var)
            all_candidates.extend(candidates)
            if candidates:
                break
            time.sleep(0.5)
    
    MIN_CANDIDATES = 3
    
    # 3. Wikipedia in priority languages
    priority_langs = []
    if country:
        # Map country to language codes
        country_lang_map = {
            'South Korea': ['ko', 'en'],
            'Japan': ['ja', 'en'],
            'China': ['zh', 'en'],
            'Brazil': ['pt', 'en'],
            'Mexico': ['es', 'en'],
            'France': ['fr', 'en'],
            'Germany': ['de', 'en'],
            # ... more mappings
        }
        priority_langs.extend(country_lang_map.get(country, ['en', 'es']))
    else:
        if detected_lang:
            priority_langs.append(detected_lang)
        priority_langs.append('en')
    
    priority_langs = list(dict.fromkeys(priority_langs))  # Remove duplicates
    
    if len(all_candidates) < MIN_CANDIDATES:
        for var in variations[:1]:
            for lang in priority_langs:
                candidates = search_wikipedia_infobox_genre_cached(var, lang)
                all_candidates.extend(candidates)
                if len(all_candidates) >= MIN_CANDIDATES:
                    break
                if not candidates:
                    candidates = search_wikipedia_summary_genre_cached(var, lang)
                    all_candidates.extend(candidates)
                if len(all_candidates) >= MIN_CANDIDATES:
                    break
                time.sleep(0.3)
            if len(all_candidates) >= MIN_CANDIDATES:
                break
    
    # 4. DeepSeek fallback
    if not all_candidates:
        logger.debug(f"  🔍 Using DeepSeek fallback for genre: {artist}")
        _, deepseek_genre, _ = search_deepseek_fallback(artist, context_country=country)
        if deepseek_genre:
            macro, _ = normalize_genre(deepseek_genre)
            if macro:
                return macro, f"DeepSeek API (genre: {deepseek_genre})"
            else:
                return deepseek_genre, f"DeepSeek API (raw: {deepseek_genre})"
    
    # 5. Country fallback
    if not all_candidates and country:
        priority = COUNTRY_GENRE_PRIORITY.get(country, DEFAULT_GENRE_PRIORITY)
        if priority:
            return priority[0], f"Genre: country fallback ({country})"
    
    if all_candidates:
        primary_genre = select_primary_genre(artist, all_candidates, country, detected_lang)
        sources = set(s for _, _, s in all_candidates)
        return primary_genre, f"Genre: {', '.join(sources)}"
    
    return None, "Genre not found"
```

## 🐛 Troubleshooting

### Common Issues and Solutions

| Error                                                    | Likely Cause                              | Solution                                                     |
| :------------------------------------------------------- | :---------------------------------------- | :----------------------------------------------------------- |
| `No chart databases found`                               | Script 1 hasn't run yet                   | Run `1_download.py` first                                    |
| `ModuleNotFoundError: No module named 'build_artist_db'` | Incorrect execution path                  | Run from project root: `python scripts/2_1.build_artist_db.py` |
| `DeepSeek API key not set`                               | Missing API key                           | Set `DEEPSEEK_API_KEY` env var (optional, script continues)  |
| `MusicBrainz rate limiting`                              | Too many requests                         | Script includes delays; be patient or increase `RETRY_DELAY` |
| `Wikipedia page not found`                               | Artist name spelling or non-existent page | Check artist name; script falls back to other sources        |
| `API timeout`                                            | Slow network or API issues                | Increase `DEFAULT_TIMEOUT` in config                         |
| `Database locked`                                        | Concurrent access                         | Wait and retry; script uses short transactions               |
| `Artist not found in any source`                         | Very obscure artist                       | Add fallback rules or use DeepSeek (if key configured)       |

2. **Propose improvements** with concrete use cases
3. **Add new genre mappings** with examples of artists that use them
4. **Contribute country variants** (especially for underrepresented regions)
5. **Improve Wikipedia pattern matching** for better extraction
6. **Maintain backward compatibility** with the existing database schema

### Quick Contribution Guide

```bash
# 1. Fork the repository
# 2. Clone your fork
git clone https://github.com/your-username/Music-Charts-Intelligence
cd Music-Charts-Intelligence

# 3. Create a branch for your contribution
git checkout -b feature/new-genre-mapping

# 4. Make changes to dictionaries or logic
# Example: Add new genre mapping
echo "('new subgenre'): ('Macro-Genre', 'subgenre')," >> scripts/build_artist_db/dictionaries/genres.py

# 5. Test locally
python scripts/2_1.build_artist_db.py

# 6. Commit with a descriptive message
git commit -m "Add genre mapping for 'new subgenre' → Macro-Genre"

# 7. Push and create Pull Request
git push origin feature/new-genre-mapping
```

## 🧪 Known Limitations and Future Improvements

### Current Limitations

- **API Dependency**: System relies on external services that may change or rate-limit
- **New Artists**: Recently emerging artists may not appear in knowledge bases
- **Niche Genres**: Some micro-genres may not have mappings yet
- **Script Detection**: Heuristic-based, may occasionally misidentify
- **DeepSeek Cost**: While minimal (~$0.002 per 100 artists), requires API key
- **Wikipedia Rate Limits**: Can be slow for bulk processing
- **Collaboration Resolution**: Complex collaborations (5+ artists) may not resolve accurately

### Known Edge Cases

| Case                                              | Current Behavior              | Proposed Improvement               |
| :------------------------------------------------ | :---------------------------- | :--------------------------------- |
| K-Pop groups with foreign members                 | Assigned to South Korea       | Detect individual member countries |
| Brazilian MCs                                     | May fall back to Sertanejo    | Prioritize Funk Brasileiro         |
| Latin collaborations with lead from other country | May yield Multi-country       | Analyze order of mention           |
| Artists with multiple nationalities               | First detected country wins   | Weight by prominence/activity      |
| Obscure regional genres                           | Mapped to generic macro-genre | Add more specific mappings         |

### Future Improvements
