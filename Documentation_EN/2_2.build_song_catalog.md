## 📋 General Description

This script is the **second component of Phase 2** in the YouTube Charts intelligence system. It builds and maintains a **canonical song catalog** by extracting distinct (artist, track) pairs from weekly chart databases and storing them in a deduplicated, incrementally growing SQLite database.

**Key enhancement:** The script now integrates the **collaboration weighting system** (previously in Script 3) to resolve country and macro-genre for each song **once**, at catalog insertion time. This eliminates redundant processing across multiple chart appearances.

### Key Features

- **Idempotent ETL**: Safe to run multiple times; never creates duplicate records
- **One-Time Country/Genre Resolution**: Each song is processed exactly once using the collaboration weight algorithm
- **Auto-incrementing Surrogate Keys**: Sequential IDs for referential integrity
- **Natural Key Deduplication**: Uses `(artist_names, track_name)` composite key for uniqueness
- **Artist Metadata Integration**: Resolves countries and genres from `artist_countries_genres.db` (Script 2.1)
- **Collaboration Weighting Algorithm**: Intelligently determines country and genre for multi-artist tracks
- **Historical Record Repair**: Automatically backfills missing country/genre data for existing catalog entries
- **Automatic Source Detection**: Finds the most recent weekly database by ISO week identifier
- **Progress Reporting**: Shows insertion progress at 25%, 50%, 75%, and 100%
- **Read-Only Source Connection**: Prevents accidental modification of source data
- **Catalog Statistics**: Displays growth metrics, country/genre distribution, and top artists
- **CI/CD Optimized**: Zero external dependencies; uses only Python standard library

### **Diagram 3: Collaboration Weighting System**

<img src="https://raw.githubusercontent.com/adroguetth/Music-Charts-Intelligence/main/Documentation_EN/Diagrams/2_2.build_song_catalog/3.png" alt="Diagram 3: Collaboration Weighting System" width="500">

This diagram shows the **intelligent decision engine** for multi-artist tracks:

1. **Input**: Receives list of artists with their countries and genres from the lookup dictionary

2. **Single Artist Check**: If only 1 artist → returns that artist's country and genre immediately ✅

3. **Filter Known Artists**: Remove artists with NULL/Unknown country

4. **Count Countries**: Calculate frequency of each country among known artists

5. **Apply Decision Rules**:

   | Rule                                    | Condition                       | Result                                               |
   | :-------------------------------------- | :------------------------------ | :--------------------------------------------------- |
   | **Rule 1 – Absolute Majority**          | >50% from same country          | Return majority country + infer genre from hierarchy |
   | **Rule 2 – Exact 50/50 Split**          | =50% with exactly 2 countries   | Return majority country + infer genre                |
   | **Rule 3 – Relative Majority (<50%)**   | Same continent and ≤2 countries | Return majority country + infer genre                |
   | **Rule 4 – Fallback (multi-continent)** | <50% with different continents  | Return "Multi-country" + "Multi-genre"               |
   | **Rule 5 – No known artists**           | All countries NULL/Unknown      | Return "Unknown" + "Pop"                             |

6. **Genre Inference** (`infer_genre_by_country`):
   - Retrieves country's genre hierarchy from `GENRE_HIERARCHY`
   - If a single genre has >50% among artists → use it
   - Otherwise, pick the highest-ranked genre present in known genres
   - Fallback to hierarchy's first genre

7. **Output**: Returns final `(country, genre)` tuple for the track

### **Diagram 4: Idempotent Insertion Logic with Country/Genre Resolution**

<img src="https://raw.githubusercontent.com/adroguetth/Music-Charts-Intelligence/main/Documentation_EN/Diagrams/2_2.build_song_catalog/4.png" alt="Diagram 4: Idempotent Insertion Logic with Country/Genre Resolution" width="500">

This diagram details the **enhanced deduplication strategy**:

1. **Start**: Receive `artist_names` and `track_name` from source
2. **Existence Check**: Query `artist_track` table for matching natural key
3. **If Record Exists**:
   - Check if `artist_country` or `macro_genre` is NULL/Unknown
   - If incomplete → **UPDATE** with resolved values (historical repair)
   - If complete → **SKIP** (increment skipped counter)
4. **If Record Not Exists**:
   - Parse artist list using delimiters (`&`, `feat.`, `ft.`, etc.)
   - Lookup each artist in `artist_lookup` dictionary
   - Apply `resolve_country_and_genre()` algorithm
   - **INSERT** new record with resolved values
5. **After All Records**: Commit transaction and report statistics

**Idempotency Guarantee**: Running the script multiple times with the same source produces the same final catalog state (no duplicates, only one-time resolution).

```python
def needs_country_genre_update(row: Optional[Tuple]) -> bool:
    """Check if record requires country/genre resolution."""
    if row is None:
        return True
    country, genre = row
    invalid_country = country is None or country.strip() == "" or country == "Unknown"
    invalid_genre = genre is None or genre.strip() == "" or genre == "Unknown"
    return invalid_country or invalid_genre
```

## 🐛 Troubleshooting

### Common Issues and Solutions

| Error                              | Likely Cause              | Solution                                                |
| :--------------------------------- | :------------------------ | :------------------------------------------------------ |
| `Artist database not found`        | Script 2.1 hasn't run     | Run `2_1.artist_countries_genres.py` first              |
| `Source directory does not exist`  | Script 1 hasn't run       | Run `1_download.py` first                               |
| `No database files found`          | No weekly databases       | Check `charts_archive/1_download-chart/databases/`      |
| `Missing 'chart_data' table`       | Corrupted source database | Re-run Script 1 for that week                           |
| `Database is locked`               | Concurrent access         | Wait or restart; no long-running locks                  |
| `sqlite3.OperationalError`         | File permissions          | Check write permissions on target directory             |
| `NULL artist_country after repair` | Artist not in metadata DB | Update `artist_countries_genres.db` with missing artist |

## 🧪 Known Limitations

- **No Indexes**: No indexes on `artist_country` or `macro_genre` for faster analytical queries (acceptable for current scale)
- **Single Source Processing**: Processes only the most recent database per run
- **No Incremental Mode**: Always processes all records from source (not just new weeks)
- **Artist Name Matching**: Exact string normalization only; no fuzzy matching for misspellings
- **Genre Hierarchy Dependency**: Accuracy depends on completeness of `GENRE_HIERARCHY` for each country
